# 要件定義書 — NudgeMe

## 1. プロジェクト概要

NudgeMe は優柔不断なユーザーを AI（Claude Haiku）がナッジして意思決定を助ける Web アプリ。
行動経済学の「ナッジ」概念をベースに、ユーザーの性格タイプに合わせた
AIキャラクターが最適な選択を提示する。

---

## 2. 機能要件

### 2.1 認証

- FR-01: メールアドレス・パスワードでユーザー登録できる
- FR-02: 登録済みのメールアドレス・パスワードでログインできる
- FR-03: ログアウトできる（JWTトークンをクライアント側で破棄）

### 2.2 性格診断

- FR-04: 初回ログイン時に性格診断クイズを受ける（スキップ不可）
- FR-05: 診断結果（性格タイプ）がアカウントに保存される
- FR-06: 後から性格を再診断できる

性格タイプ一覧:
| タイプ | 説明 |
|--------|------|
| analytical | 分析型・論理的に判断したい |
| spontaneous | 直感型・ぱっと決めたい |
| empathetic | 共感型・周りの気持ちを考える |
| competitive | 競争型・最善の選択をしたい |

### 2.3 意思決定

- FR-07: 問題文と2〜4つの選択肢を入力できる
- FR-08: AIキャラクター（毒舌 / 優しい / 体育会系）を選択できる
- FR-09: AIが選択肢から1つを選んで理由付きで表示する
- FR-10: ランダム決断ボタンで選択肢をランダムに選べる
- FR-11: 決断履歴を一覧で確認できる
- FR-12: 決断に「後悔した / しなかった」のフィードバックを記録できる

AIキャラクター一覧:
| キャラ | ID | 説明 |
|--------|-----|------|
| 毒舌 | sarcastic | ズバッと辛口で背中を押す |
| 優しい | kind | 温かく寄り添って提案する |
| 体育会系 | sporty | 前向きで熱血な決断をする |

### 2.4 統計（拡張機能）

- FR-13: 同じ質問に対して他ユーザーがどの選択肢を選んだかの統計を見られる

---

## 3. 非機能要件

- NFR-01: JWT認証（HS256）によるセキュリティ
- NFR-02: APIレスポンスタイムは通常3秒以内（AI応答は除く）
- NFR-03: TypeScript strict モードを有効化する
- NFR-04: CORS設定でフロントエンドオリジンのみを許可する
- NFR-05: DynamoDB へのアクセスは IAM ロール経由で行い、アクセスキーをコードに埋め込まない
- NFR-06: Terraform で AWS リソースをコードとして管理する
